------------------------------------------------------------
1) Form Templates
------------------------------------------------------------
Create template
POST /form-templates
Body:
{
  "name": "Phase5 Browser Form Template",
  "org_id": "11111111-1111-4111-8111-111111111111",
  "status": "draft"
}

List templates
GET /form-templates?page=1&pageSize=10

Get template
GET /form-templates/{templateId}

------------------------------------------------------------
2) Template Versions
------------------------------------------------------------
Create version
POST /form-templates/{templateId}/versions
Body:
{
  "status": "draft"
}

List versions
GET /form-templates/{templateId}/versions

Get version
GET /form-template-versions/{versionId}

Update version status (draft <-> review_pending)
PATCH /form-template-versions/{versionId}
Body:
{
  "status": "review_pending"
}

------------------------------------------------------------
3) Form Fields in Draft Version
------------------------------------------------------------
Add field
POST /form-template-versions/{versionId}/fields
Body:
{
  "field_catalog_id": "30000000-0000-4000-8000-000000000040",
  "label": "Case Notes",
  "input_type": "textarea",
  "is_required": false
}

Update field
PATCH /form-fields/{fieldId}
Body:
{
  "label": "Case Notes Updated"
}

Delete field
DELETE /form-fields/{fieldId}

------------------------------------------------------------
4) Publish Flow
------------------------------------------------------------
Publish version
POST /form-template-versions/{versionId}/publish

Expected behavior:
- Target version -> status=published, isPublished=true
- Previous published version (same template) -> archived, isPublished=false
- Template status -> active

------------------------------------------------------------
5) Create Template From Extraction
------------------------------------------------------------
Create draft template from extracted document fields
POST /form-templates/from-document/{documentId}
Body:
{
  "name": "Template From Document Extraction"
}

Expected behavior:
- New form_template (draft)
- New version_no=1 (draft)
- Form fields auto-created from mapped/extracted fields

------------------------------------------------------------
PowerShell smoke example
------------------------------------------------------------
$headers = @{
  "x-mock-user-id"="10000000-0000-4000-8000-000000000002"
  "x-mock-org-id"="11111111-1111-4111-8111-111111111111"
  "x-mock-role"="ngo_admin"
  "x-mock-firebase-uid"="firebase-ngo-admin-a-001"
}

$templateBody = @{
  name = "Phase5 Smoke Template"
} | ConvertTo-Json

$template = Invoke-RestMethod -Method Post -Uri "http://localhost:8080/api/form-templates" -ContentType "application/json" -Headers $headers -Body $templateBody

$versionBody = @{
  status = "draft"
} | ConvertTo-Json

$version = Invoke-RestMethod -Method Post -Uri "http://localhost:8080/api/form-templates/$($template.data.id)/versions" -ContentType "application/json" -Headers $headers -Body $versionBody

$fieldBody = @{
  field_catalog_id = "30000000-0000-4000-8000-000000000040"
  label = "Case Notes"
  input_type = "textarea"
  is_required = $false
} | ConvertTo-Json

$field = Invoke-RestMethod -Method Post -Uri "http://localhost:8080/api/form-template-versions/$($version.data.id)/fields" -ContentType "application/json" -Headers $headers -Body $fieldBody

$publish = Invoke-RestMethod -Method Post -Uri "http://localhost:8080/api/form-template-versions/$($version.data.id)/publish" -Headers $headers

$template
$version
$field
$publish
